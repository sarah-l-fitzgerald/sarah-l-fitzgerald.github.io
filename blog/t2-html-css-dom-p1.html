    HTML, CSS and the DOM

    The Document Object Model is a model is an application programming interface for valid and well formed HTML and XML documents. It defines the logical structure of the documents and the way the document is accessed and manipulated.

    However unless you are a techie this explanation is going to make as much sense as a recipe in Latvian (for a non Latvian readers).

    The best analogy I can find for explaining the DOM is to think of it as the remote control that sits between you and your TV.

    You know that pressing 1 on your remote will turn your TV to channel 1, or that the volume up button will increase the voulme. If you call up the TV schedule you can see what is on next.

    In this way you can understand that the DOM as the remote, you the person as a programming language and the TV as the web browser - the web pages can be thought of as any of the TV menus.

    It doesn't matter what language you speak; as long as someone presses the right button on the remote, the tv will behave in the same way.

    As with the DOM it doesn't matter what language you program with as long as you use the standard properties and methods the results will be the same.

    Just as our remote say for example a sky remote allows us to call up our planner menu- it also allows us to navigate through that menu, we can add a program to our planner or a whole series etc, just as the DOM allows you to interact and change elements of a document.

    How do HTML and CSS relate to the DOM using this analogy. Well think of HTML as the basic structure on a TV menu eg on a sky remote you would find the following menus planner, guide, on demand etc. Once you are inside one of these menus the DOM allows you to interact with different parts of it.

    Finally CSS is what makes the menus look pretty or not and easy to read or not.

    Boxifying Design

    Boxifying Design is essentially looking at the different elements within a page as a series of boxes.

    The Box Model

    The box model is a set of rules that define how every web page is rendered on the internet. CSS treats each element in your document as a box with a whole lot of different properties that determine where it appears on the page.
